Structures & routing rules

Each structure can accept ore-bearing buybacks, non-ore buybacks, and/or moon contracts. Buyback contracts route by item category; moon contracts only need docking access (any structure with the "Moon" checkbox).

Moon contracts

Moon contracts (item_exchange, price=0) get two figures: a Janice appraisal of the compressed contents at the chosen hub (informational), and the refined-mineral payout used to pay the contractor. Refined payout = refined value × payout fraction.

Moon ore covers R4–R64 (raw + compressed + graded variants). Defaults: 80% moon, 90% non-moon & ice.

NLDF alliance quota sync

Pull quotas from a single source of truth — a private GitHub repo holding a quotas.json file. Paste the URL of the file (https://github.com/<owner>/<repo>/blob/<branch>/quotas.json). Every alliance member fills the Read PAT with a fine-grained token that has Contents: read on this repo. The admin additionally fills the Write PAT + ticks Allow push and uses the Push to repo button to commit changes. Sync also accepts the raw URL or the Contents API URL of the same file.

Market history archive

Accumulates one gzipped full-depth snapshot of the Market tab's structure per day into a dedicated private GitHub repo — building the history ESI doesn't expose for player structures. Paste the repo's HTTPS URL (defaults to the main branch); the per-day path is generated automatically. The push is opportunistic (fires when the Market tab is open, at most once / 24h) and idempotent (one file per day). Use fine-grained PATs scoped to this repo only. Both PATs are opt-in at export time (Backup & share below).

Stockpile page — industry material stock

The Stockpile tab (in the Operations / Admin section) shows alliance mineral / PI / material stock levels, stored as inventory/stock.json in the Market history repo above. Access is password-protected — enter the password below to unlock the tab (remembered on this machine only; the password is not saved or exported with your config). Tick Allow stock edits to unhide the admin paste/save panel; editing further requires Industry Officer / Acquisitions Officer group membership and the Market-history Write PAT.

Backup & share

Download a JSON file of every setting on this tab — corp ID, structures, refining settings, mail presets, home structure, quotas. Useful for backing up before a re-install, or sharing a baseline config with another admin. ESI tokens are not exported (they stay in tokens.json); the Janice API key is included by default but can be left out at export time.

SRP — Ship Replacement payouts

Pulls outstanding SRP requests from auth.navaldefence.org/srp/ (uses the Alliance Auth login from the Doctrines tab) and computes a recommended payout per the SRP scheme: payout = rate × the killboard loss value. Each request's category is detected from the killmail's fitted modules — command bursts ⇒ Links, remote reps ⇒ Logistics (100%), interdictors ⇒ 100% — so links T3D/Nighthawk and T1-cruiser logi are recognised by what they actually fitted, not the hull name; everything else defaults to Standard 75%. Each hull is also checked against the Auth doctrine list (a ⚠ not in doctrine flag shows when it isn't). Use each row's dropdown to set Fight Club (max 10M) / Hisec-NPC (max 5M) or correct a guess — the recommended figure updates live and is click-to-copy, as are pilot names. Eligibility (what qualifies) stays your call; this only does the math.
